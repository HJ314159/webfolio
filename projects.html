To-Do List

GitHub link: https://github.com/HJ314159/to-do-list

CodePen link: https://codepen.io/rollingorange/pen/wvNdgZG

Description

To-do list is a simple web application designed for managing tasks. It offers the following features:

  • Adding tasks
  • Deleting tasks
  • Changing the theme of the website

To-do list allows users to add tasks and automatically saves them in local storage, ensuring that users can access their tasks even if the browser is closed.

Users can mark tasks as "done" to track their progress effectively.

The application provides a choice of six themes, including the default white theme, blue, green, orange, grey, pink

Technical Stack

  • HTML
  • CSS
  • JavaScript
  • Visual Studio Code (Code Editor)
  • GitHub (Version Control)

Challenges Faced

During the development of this to-do list project, several challenges were encountered:

  • Ensuring the web application's responsiveness on various devices.
  • Integrating local storage to save and retrieve tasks, making them accessible even after browser sessions.

This project taught me a bunch about making things look good on any screen, and it was pretty cool to learn.

Future Goals

The to-do list project was a fantastic way for me to hone my web development skills and dive into creating a practical application. While there won't be any further updates to this particular project, it's just the beginning of my journey. My goal is to tackle more complex web applications in the future, each one pushing the boundaries of what I can do with code.

Tic-Tac-Toe Game

GitHub link: https://github.com/HJ314159/web-based-tic-tac-toe-game

Description

This web-based tic-tac-toe game allows users to enjoy a classic game with a modern twist. It includes the following features:

  • Interactive gameplay with two players taking turns.
  • Win detection to determine the game's outcome.
  • Stylish color palette and fonts for an engaging visual experience.

Tech Stack

  • HTML
  • CSS
  • Vanilla JavaScript
  • VS Code (Code Editor)
  • GitHub (Version Control)

Challenges Faced

During the project's development, I came across some interesting challenges:

  • Managing the game state and ensuring that it accurately reflects the current state of the game while preventing players from making additional moves after the game has ended.
  • Displaying the actual game state with icons
  • Deciding on what the layout of the website would be

Fortunately, I managed to resolve all these issues (albeit with some frustrating moments) to shape the project into what it is today.

Future Goals

One exciting addition I've got in mind is implementing a formidable AI opponent. This feature will level up the single-player mode, adding a fun challenge while allowing me to delve into AI and machine learning within the realm of web development. I envision an AI opponent that can adapt to the player's moves and provide a truly immersive gaming experience.

Of course, I can't deny that there's a learning curve ahead. However, I'm genuinely enthusiastic about embarking on this learning journey and incorporating this AI feature as it combines both of my interests. It's an opportunity to expand my skill set, and I'm looking forward to tackling the challenges that come with it.
